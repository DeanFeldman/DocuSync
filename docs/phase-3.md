# Phase 3 Windows desktop integration

## Objective

DocSync now ships the Phase 2 controlled DOCX-editing workflow as its primary Windows desktop experience. Electron opens the application, starts a bundled FastAPI/Python document service automatically, and displays the React workspace without requiring a browser, Python, Node.js, or developer commands from the installed user.

The earlier named-placeholder plain-text prototype remains in `apps/desktop-ui/` and `apps/template-api/` as technical evidence. It is not included in the 1.0.0 installer.

## Installed workflow

1. Open DocSync from its Windows application window.
2. Upload 2–20 related DOCX files as a document set.
3. View the selected file through a PDF generated by Microsoft Word, preserving Word pagination and layout.
4. Switch to the structured Select text view and choose a supported paragraph, heading, or list item.
5. Review exact matches across the related files and explicitly include or exclude each target.
6. Preview the complete before/after impact.
7. Apply the edit to new immutable DOCX versions and continue editing those current versions.
8. When every edit is finished, use the fixed bottom action to save the final ZIP through Electron’s Windows download dialog.

If Microsoft Word is unavailable or cannot render a file, the app falls back to the clearly labelled structured preview rather than pretending to reproduce Word layout.

## Component boundaries

| Component | Location | Responsibility |
|---|---|---|
| Desktop shell | `apps/desktop/` | Window and backend lifecycle, session cookie, navigation and permission restrictions, native download dialog, packaging. |
| Frontend | `apps/web/` | Upload, Word-layout view, structured selection, target confirmation, impact review, continued editing, final download. |
| Backend | `apps/api/` | DOCX validation and storage, element extraction, exact matching, Microsoft Word rendering, immutable generation, ZIP output, SQLite audit data. |
| Frozen backend | `apps/api/desktop_backend.py`, `docsync-api.spec` | Self-contained Windows FastAPI/Python service produced by PyInstaller. |
| Retained template proof | `apps/desktop-ui/`, `apps/template-api/` | Earlier plain-text proof; tested but not shipped as the main application. |

## Run and package

```powershell
npm install
python -m pip install -r apps/api/requirements.txt
npm start
```

Build the assisted NSIS installer:

```powershell
python -m pip install -r apps/api/requirements-build.txt
npm run dist:win
```

The installer is written to `release/v1/`. PyInstaller 6.21.0 freezes the Python service and Electron Builder packages it with the React build and Electron runtime.

## Current limitations

- Microsoft Word desktop is required for Word-authored pagination and layout rendering. The installed app does not redistribute Word.
- Selection and replacement cover recognised paragraphs, headings, and list items in the main document body. Tables, headers, footers, comments, tracked changes, and text boxes are not selectable yet.
- Replacement preserves the paragraph style and first-run formatting; mixed formatting within replaced text is not guaranteed.
- Projects are stored locally and there are no accounts, cloud synchronisation, collaboration features, automatic updates, or external integrations.
- The installer is unsigned, so Windows may display an unknown-publisher warning.

## Risk register and next gates

| Risk | Mitigation | Next evidence |
|---|---|---|
| Unsupported Word structures lose fidelity during edits | Immutable originals, supported-element boundary, real Word previews | Representative complex-DOCX spike and before/after integrity checks |
| Installer works only on the developer computer | Frozen backend, Windows CI packaging, checksum | Clean Windows account or VM install/use/uninstall record |
| Another local process contacts the API | Ephemeral localhost port, random HttpOnly session cookie, request validation | Revisit threat model before sensitive production use |
| Electron and bundled Python increase installer size | Accepted for the desktop release; unused Python packages excluded | Reconsider only after user validation |
| Scope expands to PPTX before DOCX is reliable | Keep PPTX explicitly deferred | Complete representative DOCX validation first |
